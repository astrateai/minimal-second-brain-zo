# WIKI_SCHEMA.md

Purpose: maintain a dead-simple, durable second brain using only markdown files.

## Folder Contract

- `raw/` = unorganized source dump (the only folder the user writes to directly)
- `wiki/` = organized knowledge pages generated/updated by AI
- `outputs/` = answers, reports, and one-off deliverables generated by AI

Hidden operator automation may exist in `.ops/`, but end-user workflow does not change.

## Core Rules

1. Keep source truth in `raw/`.
2. Do not manually curate `wiki/` structure unless necessary.
3. Prefer updating an existing wiki page over creating a new one.
4. Every meaningful claim in `wiki/` should point to source files in `raw/`.
5. Keep writing dense and scannable.
6. Avoid plugin/tool lock-in. Markdown first.
7. End users should only be told to add material to `raw/`.

## Chat Capture Trigger Contract

Use this deterministic policy for what gets saved into the wiki pipeline:

1. If a message starts with `wiki:` -> capture that message (and attachments) into `raw/`.
2. If a message includes `wiki this` -> capture the current thread item into `raw/`.
3. If a message includes `no wiki` -> do not capture this message.
4. If a file or link is shared in chat -> capture to `raw/` by default.
5. If content is dropped into `raw/` directly -> process normally in compile.
6. Regular chat without these triggers should not be auto-captured unless the content is clearly durable.

Priority order for conflicts:

1. `no wiki` wins over everything.
2. Explicit wiki commands (`wiki:`, `wiki this`) win over normal chat behavior.
3. File/link uploads still auto-capture unless `no wiki` is present.

## File Naming

- Raw files: timestamp + short slug.
  - Example: `2026-04-10-kickoff-call-notes.md`
- Wiki pages: kebab-case concept names.
  - Example: `offer-positioning.md`
- Outputs: timestamp + intent.
  - Example: `2026-04-10-q2-growth-brief.md`

## Default AI Tasks

### Compile task
"Read all files in `raw/`. Build or update pages in `wiki/` by topic. Create `wiki/index.md` if missing. Link related pages. Keep source references to raw filenames."

### Question task
"Answer this question using only `wiki/` and direct source references from `raw/`. Save the final answer to `outputs/<filename>.md`."

### Health check task
"Review `wiki/` for contradictions, unsupported claims, stale pages, and duplicate topics. Save findings to `outputs/<date>-health-check.md`."
